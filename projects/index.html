Python Picture-to-Text

A python tool used to extract text from images. For a more detailed view, see the repository.

Problem

I enjoy the thrive to the paperless world, as I find it much easier organizing digital documents. But since many documents are still required as a hardcopy, one is still forced to archive those.

Solution

In order to reduce the amount of time spent organizing those documents, I bought a numbering stamp. This stamp automatically increases the number with each stamping. It is put on each document. Then I take a picture of all documents. After that, the pictures are organized in folders by the name of the number on the document. Then a python script is started with the path to the images. This script takes one to many pictures per document, tries to align them in a way that the text is “as horizontal as possible”. This is done, since one easily is off by a few degrees when taking pictures with a camera or a phone. Then the slightly rotated pictures are stored to a pdf and used to extract text from them. This text then is stored to txt-files. The hardcopy documents are now simply archived in order of the number of the stamp. Whenever I now need a certain document, I can simply search for it on the computer and it will tell me, which number the document has.

Drawback

The image must have a good resolution and the text must be well readable, otherwise the output will not be all that great. In that case, it is possible that searching fails. But it is possible to add descriptive names to the number of the folder name through which manual searching can be improved.

Used technologies

  • Python 3
  • PyOcr
  • Img2pdf
  • Opencv_python
  • Tesseract

Weatherstation

Tracks the room temperature and humidity. The stored data is presented in a local webpage. For a more detailed view, see the client repository and the server repository.

Idea

Trying to find a project, which (at least) touches several technologies, I thought to create a weather station for tracking the room climate. The main goal behind this idea was to get a better insight in unknown technologies.

Setup

The project consists of two major parts: The client, which measures the current temperature and humidity, and a server, which stores the received values and presents them in a nice way.

Client

The client periodically measures the room climate and publishes the values based on MQTT. It is based on an ESP8266.

Server

The server’s task is to store and visualize the temperature data. This can either be done with the node js webserver, which was self created, or with node-red, influxdb and grafana

Node JS Server

The server stores the published data the client to a SQL database. Further, it starts a webserver, which visualizes the data in form of graphs.

Node Red, InfluxDB and Grafana

Alternatively one can also define a node red flow, which automatically stores the published data to an InfluxDB. The visualization then is done with the help of Grafana.

Used technologies

  • PlatformIO
  • ESP8266
  • NodeJS
  • SQL
  • Python
  • Node Red
  • MQTT
  • Docker / Docker-compose
  • Grafana
  • InfluxDB

Coffee Grinder Automation

When started, switches a coffee grinder on until a given time has passed, or a given target weight is reached. See the github repository for a detailed description.

Idea

I enjoy a good cup of coffee. For that, I have bought myself a coffee grinder to start with, along with a nice espresso machine. To achieve a proper coffee result, several factors are important. One of them is the bean amount. So far, I had to stand by the mill with a scale and manually measure the amount of beans, that has come out of it. After having worked from home for a while, I got annoyed of this, so I decided to automatize the grinding of the beans.

Goal

My major goals of this project were:

  • Automatically grind the beans until a certain weight is reached.
  • Use as little hardware as possible

Solution

Having gained some experience in the ESP8266 world, I decided to use a D1-mini for the task. Further, after some research, I chose to use a load cell along with the load cell amplifier HX711. The mill itself is switched on and off by a relay controlled by the D1-mini.

A node-red flow along with the node-red-dashboard are used to control the D1-mini. The commands are sent through mqtt topics.

Used technologies

  • C++
  • PlatformIO
  • ESP8266
  • Node Red
  • MQTT
  • Docker / Docker-compose
